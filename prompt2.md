## 0. 一句话愿景
用一场“悬疑式破案”的短体验，把生日祝福藏进线索里：**越接近真相，越接近温柔**。

---

## 1. 目标、范围与成功标准

### 1.1 目标
1. 做一个“悬疑味”的生日祝福网页：有剧情、有推理、有彩蛋、有最终反转。
2. 全程不依赖表妹对原剧剧情的记忆，致敬只体现在**氛围与机制**上。
3. 体验结束后留下一页“可反复回看”的纪念终章（相册/祝福/称号）。

### 1.2 非目标
- 不做账号体系、不做后台数据库、不做多人在线。
- 不追求“防作弊安全”，只做轻量“防一眼看源码”的遮挡即可。

### 1.3 成功标准（验收口径）
- 主线通关时长：**12–20 分钟**；全 6 关完整：**20–35 分钟**。
- 任一关卡：**> 3 分钟无进展**时，玩家仍能通过提示系统“体面脱困”。
- 音频：首次进入必须能通过一次点击获得播放权限；后续关卡切换不丢失。
- 体验“可控”：音量/静音/字幕/减少动效/跳过祝福彩蛋（可选）齐备。
- 体验“可恢复”：刷新/切后台/来电后，能从断点恢复。

## 2. 体验原则（从氛围到情绪曲线）

### 2.1 节奏结构
- 每关遵循：**1–2 分钟观察搜证** → **2–6 分钟推理解码** → **15–30 秒奖励彩蛋**
- 总体情绪：紧张 → 轻松一笑 → 再紧张 → 最终温柔爆发

### 2.2 悬疑与生日的融合方式
- 生日快乐歌不建议全程单曲循环；建议：
  - **关卡内**：悬疑 BGM（低干扰、便于思考）
  - **过关奖励**：祝福短音频/短视频（带“探案口吻”）
  - **终章**：生日主题曲完整响起 + 温柔画廊

### L1 审讯室的逻辑网格（致敬：大侦探）
**目标**：通过“搜证 + 逻辑网格”锁定唯一解，并得到碎片 F1。
**交互关键词**：点击搜证点、线索卡片、逻辑网格打勾/打叉、提交密码。

#### 7.1 页面布局（建议）
- 左侧：案情板（嫌疑人卡、证物卡、可点击搜证区域）
- 右侧：逻辑网格（Suspect×Room、Suspect×Item 两张表或合并表）
- 顶部：进度条、声望分、提示按钮、设置按钮

#### 7.2 谜题数据（示例可直接用）
**嫌疑人**（可换成亲友头像 + 悬疑滤镜）：
- 阿梨、波波、橙子、豆豆

**房间**：
- 厨房、客厅、书房、阳台

**证物（每件证物图上印一个数字）**：
- 面粉（数字 2）
- 打火机（数字 7）
- 手套（数字 9）
- 红线（数字 4）

**线索卡（建议以“搜证点”形式分散出现）**：
1. 书房里的人戴着手套。
2. 厨房里的人手上沾了面粉。
3. 拿打火机的人在客厅。
4. 阳台里的人拿着红线。
5. 波波没有戴手套。
6. 阿梨不在厨房，也不在客厅。
7. 橙子不在书房，也不在客厅。
8. 阿梨没有戴手套。

**唯一解（给你做校验用，不直接展示给玩家）**：
- 阿梨：阳台 / 红线
- 波波：客厅 / 打火机
- 橙子：厨房 / 面粉
- 豆豆：书房 / 手套

**通关密码生成规则**（写在 UI 小字里，避免玩家不知道怎么从“解”到“密码”）：
- “按房间顺序【厨房→客厅→书房→阳台】输入证物数字”
- 得到 F1 = **2794**

#### 7.3 提示系统文案（必须提前写好）
- Tier1（免费 + 冷却）：
  - “先把‘房间→证物’这种直接对应关系填满，会很快清爽。”
- Tier2（-5 分）：
  - “阿梨不可能在书房：书房必然是手套，但阿梨明确‘没有戴手套’。”
- Tier3（-15 分）：
  - “房间顺序的证物数字是：厨房2、客厅7、书房9、阳台4。”

#### 7.4 失败与降级
- 连错 3 次密码：红屏 0.8s + -2 分 + 弹出“要不要申请上级支援（提示）”

### L2 城市地理坐标连线（致敬：唐探）
**目标**：按暗语路线连线，得到碎片 F2。
**交互关键词**：地图缩放（可选）、点击节点、SVG 路径绘制、完成校验。

#### 7.1 两种实现（选其一）
- **轻量版（推荐）**：自制“4×4 网格霓虹地图”，节点固定，体验稳定
- **沉浸版**：重绘真实城市地图（工作量更大，移动端兼容更难）

#### 7.2 谜题数据（轻量版示例）
- 节点（用图标表示）：A 档案馆、B 夜宵摊、C 天桥、D 电影院、E 河岸、F 便利店、G 旧校门、H 地铁口
- 暗语路线（展示为一段“探案指令”）：
  1) “从旧校门出发，先去最亮的那盏灯。”
  2) “穿过天桥，沿河走到能听见回声的地方。”
  3) “最后回到地下的入口，别回头。”

> 你在实现里把这些句子对应成节点序列（例如 G→F→C→E→H），玩家要在地图上连出来。

#### 7.3 校验与产出
- 连接正确后：路径整体发光，镜头拉远，路径形状显现为一个字母/汉字
- F2（示例）：显现字母 **H**（或你想要的拼音首字母）

#### 7.4 提示文案（示例）
- Tier1： “先找‘旧校门’这个起点图标，别急着连线。”
- Tier2： “‘地下入口’指的是地铁口节点，不是档案馆。”
- Tier3： “正确顺序是：G→F→C→E→H。”

### L3 破碎记忆时间线（致敬：谁是凶手 / 痕迹）
**目标**：把 6 张碎片照片拖拽排序，得到碎片 F3。
**交互关键词**：拖拽排序、自动吸附、完成校验、照片“洗印”动画。

#### 7.1 谜题结构
- 卡片数量：5–7 张（建议 6 张）
- 每张卡片包含：
  - 一张“做旧”图片（可是真实照片的局部裁切）
  - 一个小线索（如时间、天气、物体状态：蜡烛长度、饮料冰块、天色）

#### 7.2 校验策略
- 正确顺序命中后：执行“洗印变清晰”动画 + 翻面出现祝福
- F3：一段短词（示例：**“安心”** / **“继续发光”**）

#### 7.3 提示文案（示例）
- Tier1： “注意光线：哪张更像傍晚，哪张更像夜晚？”
- Tier2： “蜡烛从长到短是单向变化，不会反着长回去。”
- Tier3： “顺序是：3→1→6→2→4→5。”

### L4 法医微观视界（致敬：法医秦明 / 模仿犯）
**目标**：用 UV 光扫出“规律符号”，再用密码轮映射成明文，得到碎片 F4。
**交互关键词**：鼠标/手指当探照灯、收集符号、旋转密码轮、解码。

#### 7.1 交互实现建议（稳）
- 背景是一页“加密报告”（大量字符）
- 指针附近加一层圆形遮罩（mix-blend-mode / mask）
- 被 UV 扫到的“目标字符”高亮并加入收集槽

#### 7.2 解码建议（降低学习成本）
- 密码轮不要太硬核：建议固定 1 次旋转（例如 shift=+3）
- F4（示例）：得到两位数字或一个短串（如 **“03”** 或 **“K3”**）

#### 7.3 提示文案（示例）
- Tier1： “不是所有发光字符都要收集，优先收集出现频率一致的那一组。”
- Tier2： “密码轮只需要转到‘锁孔发亮’的位置，不必乱试很多圈。”
- Tier3： “收集符号为：◇◆◇◆，轮盘转到 +3，明文是：K3。”

### L5 多维空间映射（致敬：摩天大厦）
**目标**：从证词里定位唯一房间，得到碎片 F5。
**交互关键词**：点击房间听证词、标注“可信/不可信”、定位交汇点、输入门牌号。

#### 7.1 建议的难度旋钮
- 新手版：没有说谎者，纯交汇定位
- 标准版：1 个说谎者（需要从矛盾中排除）
- 进阶版：2 个说谎者（不建议用于生日主线）

#### 7.2 产出
- F5：门牌号（示例：**1404**）或坐标（示例：**(4,1)**）

#### 7.3 提示文案（示例）
- Tier1： “先把每个证词‘指向哪里’用标记画出来，再找交集。”
- Tier2： “如果两条证词互相矛盾，至少有一条不可信。”
- Tier3： “可信证词交汇点指向：1404。”

### L6 漫长黑夜与最终黎明（致敬：沉默的真相）
**目标**：通过“沉默机制”触发终局，然后拼接碎片得到 Final Key。
**交互关键词**：无输入检测、倒计时、拼钥 UI、最终转场。

#### 7.1 沉默机制（可读性很重要）
- 屏幕几乎全黑 + 红色安全灯
- 文案刻在行李箱上：
  - “有些真相，只在沉默里出现。”
- 逻辑：监听 `mousemove/keydown/click/scroll/touchmove`
  - 若检测到 **连续 {12} 秒无输入** → 通过
  - 若中途有输入 → 红灯急闪 + 计时重置（但不要太“惩罚感”）

> 建议：在无输入第 3 秒时，让红灯从急促变稳定，给出“这不是卡死”的暗示。

#### 7.2 终极密钥拼接规则（务必简单明确）
- 示例规则：
  - Final Key = `F1 + "-" + F2 + "-" + F3 + "-" + F4 + "-" + F5`
  - 例如：`2794-H-安心-K3-1404`
- 若是主线模式：只拼主线碎片，支线碎片影响称号/相册解锁数量

#### 7.3 破晓转场（情绪高潮）
1. 视觉：黑 → 暖白渐变（3 秒）
2. 听觉：悬疑环境音淡出 → 生日主题音乐进入
3. 文案弹窗（由机密卷宗风切换成贺卡风）：
   - “致首席调查员：案件告破。唯一真相是——世界都在为你庆生。”

## 9. 视听系统与自动播放策略（工程必读）

### 9.1 交互闸门（Gate）
- 所有带声音的音频/视频播放必须由用户第一次点击触发：
  - “指纹解锁 / 接受委托”按钮点击事件里执行 `audio.play()`

### 9.2 分轨与淡入淡出
- 背景 BGM 与彩蛋视频独立音量
- 过关时：BGM 0.6s 淡出 → 视频播放 → 结束后 BGM 0.6s 淡入

### 9.3 资源预加载
- 下一关视频/音频在上一关后半段预加载（避免“过关卡顿毁氛围”）
- 视频建议短、码率低、可选“无声字幕版”作兜底

## 10. 素材规范与合规（避免上线前翻车）

### 10.1 版权与致敬原则
- 尽量不用直接剧照/原声/BGM；用“自制素材 + 机制致敬”最稳
- 若需要音效，优先使用可商用/可再分发的素材库或自己录制

### 10.2 格式与命名（建议）
- 图片：WebP（手机更省流）
  - `img/L1_suspect_ali.webp`
- 音频：MP3/AAC（短循环）
  - `audio/bgm_L1.mp3`
- 视频：MP4(H.264)，建议竖屏优先
  - `video/reward_L1.mp4`

## 11. 技术实现建议（收敛选型，减少返工）

### 11.1 推荐架构
- SPA + 路由（或单页面状态机）
- 状态管理：轻量即可（不必上复杂方案）
- 动画：CSS + 少量 JS（reduceMotion 一键降级）

### 11.2 移动端重点坑位清单
- 拖拽：优先 Pointer Events（比原生 DnD 更稳）
- 音视频：微信内置浏览器可能需要更谨慎的“用户手势链”
